# Kuuu リデザイン設計書

作成日: 2026-03-09

## 1. プロジェクト概要

### ビジョン
飲食店の「今この瞬間」を、近くにいる人に届ける SNS プラットフォーム。
Instagram 連携を軸に、位置情報ベースで飲食店の最新投稿を地域の人に届ける。

### 食べログ・ホットペッパーとの差別化
- 「検索して探す」ではなく「フォローしてる店の今日が見える」
- Instagram の投稿はフォロワーにしか届かない → Kuuu は**近くにいる人**に届ける
- マップは補助機能。ストーリーリングが浮かぶ SNS 的マップ

### コア体験
「好きな店の今日の様子が見える」— つながり・関係性ベースの発見

### ターゲット
- **リピーター層**: 行きつけの店の最新情報を知りたい人
- **探索層**: 新しい店を見つけたい人
- 両方を同じくらい重視する

### コンテンツ戦略
- **Phase 1（現在）**: Instagram 連携が中心。店側のハードルを最小化
- **将来**: Kuuu 独自投稿を中心に移行
- Instagram ストーリーも Graph API で取得・表示する

### デザインの方向性
- Instagram 的なクリーンさ + 実用情報（空席・距離）が一目でわかる
- 詳細なデザイン指示は Antigravity 側で行う

## 4. 技術的な決定事項

### 維持するもの（バックエンド層）
- DB スキーマ（Supabase テーブル設計）
- API Routes（認証、Stripe、Instagram OAuth）
- Supabase RLS ポリシー
- Stripe 連携（3プラン）
- Instagram Graph API 連携ロジック

### 新規作成するもの（フロントエンド層）
- デザインシステムのコンポーネント
- レイアウト（AppShell、ナビゲーション）
- 全ページの UI

### Instagram API
- 料金: **無料**（Meta Graph API）
- レート制限: 200コール/時間/ユーザー
- ストーリー: Graph API `GET /{ig-user-id}/stories` で取得可能（24時間制限あり）

### 空席ステータス
- 店舗オーナーが手動でボタンを押して切り替え
- 4段階: 空席あり / やや混雑 / 満席 / 本日休業
- Supabase Realtime でリアルタイム反映

### 予約
- スタンダードプラン以上の店舗のみ
- 予約フロー: 日時・人数入力 → 確認 → 送信 → 店舗が確定/キャンセル

### デザイン詳細指示
- Antigravity 側で行う
