# モグリス フィードアルゴリズム ロードマップ

## コア思想
モグリス のアルゴリズムの核心は **「フォローしていなくても、近くにいるだけで店舗の投稿が届く」** こと。
Instagram ではフォロワーにしか届かないが、モグリス では地域の全ユーザーに届く。これが最大の差別化ポイント。

## 実装済み（Phase 0〜3）— 10要素スコアリング

### Phase 0: 基本スコアリング
| 要素 | 近くタブ | 人気タブ | 説明 |
|------|---------|---------|------|
| 距離 | 主要（5km以内） | 軽い減衰（10km+） | Haversine 距離計算 |
| 鮮度 | ○ | ○ | 24h以内=1.0、7日で0.1に減衰 |
| 空席ボーナス | ○ | — | available=1.5, busy=1.1, full=0.8, closed=0.3 |

### Phase 1: パーソナライズ基礎
| 要素 | 説明 |
|------|------|
| 時間帯最適化 | ランチ帯→ラーメン系1.4倍、ディナー帯→居酒屋系1.4倍、カフェタイム→カフェ系1.3倍 |
| 閲覧済み減衰 | 表示済み投稿のスコアを0.3倍に（localStorage、7日間保持、上限200件） |
| 多様性制御 | 同じ店舗の投稿が連続しないよう最低2つ間隔を確保（`diversifyPosts`） |

### Phase 2: 学習 + コンテキスト
| 要素 | 説明 |
|------|------|
| ジャンル嗜好学習 | クリック履歴から好みのジャンルを学習、最大1.5倍ブースト（localStorage、30日間） |
| 営業時間フィルター | 営業中=1.0、開店1h前=0.7、時間外=0.4、定休日=0.2（深夜営業対応済み） |

### Phase 3: エンゲージメント + 品質
| 要素 | 説明 |
|------|------|
| トレンディング | 直近6hのいいね+フォロー勢いで最大1.8倍ブースト |
| 新店舗ブースト | 登録3日以内=1.6倍、14日以内=1.3倍 |
| 投稿品質 | 画像なし=0.5倍、キャプション100文字+=1.3倍、50文字+=1.2倍、なし=0.8倍 |

### 関連ファイル
- `src/lib/feed-scoring.ts` — スコアリングロジック本体
- `src/lib/geo.ts` — Haversine 距離計算 + フォーマット
- `src/app/api/feed/nearby/route.ts` — 近くタブ API
- `src/app/api/feed/popular/route.ts` — 人気タブ API
- `src/app/(main)/home-client.tsx` — クライアント側（閲覧トラッキング、ジャンル嗜好）

## 未実装（ユーザー数増加後に実装）

### Phase 4: 協調フィルタリング（ユーザー数 1,000+ で実装）
- **「AをフォローしたユーザーはBもフォロー」** パターンを検出
- ユーザー間の類似度を計算し、似たユーザーが好む店舗を推薦
- 必要テーブル: `user_similarities`（定期バッチ計算）
- 実装方法: コサイン類似度 or Jaccard係数でフォロー重複を計測

### Phase 5: 行動シグナル強化（MAU 5,000+ で実装）
- **滞在時間トラッキング**: 投稿詳細ページの閲覧秒数を記録
  - 3秒未満: 興味なしシグナル → スコア減衰
  - 10秒以上: 興味ありシグナル → ジャンル嗜好に加算
- **スクロール深度**: フィードのどこまでスクロールしたかを記録
- **ネガティブシグナル**: 「興味なし」ボタンの追加
  - 特定店舗/ジャンルの非表示リクエスト
  - 必要テーブル: `user_negative_signals`

### Phase 6: A/Bテスト基盤（MAU 10,000+ で実装）
- スコアリング係数の動的最適化
- ユーザーをグループ分けし、異なるパラメータで配信
- コンバージョン率（表示→クリック→予約）をKPIとして自動チューニング
- 必要: `ab_test_groups`, `ab_test_results` テーブル

### Phase 7: 高度な最適化（MAU 50,000+ で実装）
- **リアルタイムパーソナライゼーション**: セッション内の行動で即時スコア調整
- **季節・天候連動**: 天気API連携（雨の日→屋内店舗優先）
- **予約空き状況連動**: 予約枠が残り少ない店舗のリアルタイムブースト
- **機械学習モデル**: クリック予測モデルの導入（特徴量: 全スコアリング要素）

## スコアリング数式（現在）

### 近くタブ
```
score = 距離 × 鮮度 × 空席 × 時間帯 × 閲覧済み × ジャンル嗜好 × 営業時間 × トレンディング × 新店舗 × 品質
```

### 人気タブ
```
score = エンゲージメント × 鮮度 × 距離減衰 × 時間帯 × 閲覧済み × ジャンル嗜好 × 営業時間 × トレンディング × 新店舗 × 品質

エンゲージメント = フォロワー数×2 + いいね数 + 予約数×3
```
